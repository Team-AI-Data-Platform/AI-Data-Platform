# 전자금융 장애 대응 매뉴얼

## 1. 목적

이 문서는 전자금융 서비스 장애 발생 시 초기 점검 절차와 담당자 역할을 정의한다.

## 2. 초기 확인 항목

전자금융 장애가 접수되면 다음 항목을 우선 확인한다.

1. API Gateway 상태
2. 인증 서버 상태
3. 계좌조회, 이체, 로그인 등 주요 거래 오류율
4. WAS 및 DB 연결 상태
5. 외부 연계 기관 응답 상태

## 3. 장애 대응 절차

장애 발생 시 운영 담당자는 모니터링 대시보드에서 오류율과 응답시간을 확인한다.
개발 담당자는 최근 배포 이력과 로그를 확인한다.
인프라 담당자는 서버, 네트워크, DB 상태를 점검한다.

## 4. 보고 기준

고객 영향이 있는 장애는 10분 이내 1차 보고를 수행한다.
장애 원인과 조치 계획은 확인되는 즉시 공유한다.
