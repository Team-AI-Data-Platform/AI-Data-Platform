    Local LLM 모델 크기(1B, 4B, 8B, 14B, 20B, 32B) 이해하기

    개요

    Local LLM을 공부하다 보면 gemma3:4b, qwen3:8b, qwen3:14b 같은 모델명을 보게 됩니다.

    여기서 B는 Billion(10억)을 의미합니다.

    표기 의미
    1B 10억 개 파라미터
    4B 40억 개 파라미터
    8B 80억 개 파라미터
    14B 140억 개 파라미터
    20B 200억 개 파라미터
    32B 320억 개 파라미터
    70B 700억 개 파라미터

    파라미터(Parameter)란?

    LLM의 파라미터는 모델이 학습을 통해 습득한 지식과 패턴을 저장하는 값입니다.

    쉽게 말하면 사람의 뇌 속 신경망 연결 정보와 비슷한 개념입니다.

    파라미터가 많을수록:

    • 더 많은 지식을 저장 가능
    • 더 복잡한 추론 가능
    • 더 자연스러운 답변 가능
    • 더 긴 문맥 이해 가능

    반면:

    • 메모리 사용량 증가
    • 모델 크기 증가
    • 응답 속도 저하 가능

    개발자 관점의 체감

    모델 크기 체감 수준
    1B~4B 초급 개발자
    7B~14B 경력 개발자
    20B~32B 수석 개발자
    70B 이상 전문 컨설턴트 수준

    Spring Boot 질문 예시

    질문:

    Spring Cloud Gateway와 WebFlux를 사용하여 API Gateway를 구성하는 방법은?

    4B 모델

    기초적인 설명 위주

    8B 모델

    설계 방향 설명 가능

    14B 모델

    실무 수준 설명 가능

    32B 모델

    아키텍처 수준 논의 가능

    모델이 커질수록 필요한 자원

    항목 변화
    다운로드 시간 증가
    디스크 사용량 증가
    메모리 사용량 증가
    GPU 사용량 증가
    답변 품질 향상

    장관님 장비 기준

    Mac Mini M2 Pro 32GB

    권장:

    • gemma3:4b
    • qwen3:8b
    • qwen3:14b
    • qwen3:32b

    Windows Ultra 7 255H / 32GB

    권장:

    • gemma3:4b
    • qwen3:8b
    • qwen3:14b

    Step 1 학습 순서

    ollama run gemma3:4b

    ollama run qwen3:8b

    ollama run qwen3:14b

    이후 Open WebUI를 연동하여 브라우저 기반 환경을 구성합니다.

    결론

    AI 플랫폼 아키텍트 학습의 핵심은 가장 큰 모델을 사용하는 것이 아니라 Local LLM의 전체 구조를 이해하는 것입니다.
