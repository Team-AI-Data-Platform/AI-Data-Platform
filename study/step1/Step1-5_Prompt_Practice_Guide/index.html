    Step1-5 Prompt 실습 가이드

    문서 목적

    본 문서는 Local LLM 구축 Step1 과정 중 Prompt 실습을 위한 가이드입니다.

    이전 단계에서 우리는 다음 작업을 완료했습니다.

    Step1-1 Ollama 설치
Step1-2 Local LLM 모델 실행
Step1-3 Python 가상환경과 Ollama 연동
Step1-4 Open WebUI 구축

    이제부터는 Open WebUI 또는 Ollama API를 통해 Local LLM에게 질문을 잘 전달하는 방법을 연습합니다.

    Prompt 실습의 목적은 단순히 질문을 던지는 것이 아니라, LLM이 원하는 방식으로 답변하도록 지시하는 방법을 익히는 것입니다.

    1. Prompt란 무엇인가?

    Prompt는 LLM에게 전달하는 입력 문장입니다.

    쉽게 말하면 다음과 같습니다.

    Prompt = AI에게 주는 지시문

    예:

    Local LLM이 무엇인지 초보자에게 설명해줘.

    이 문장이 바로 Prompt입니다.

    LLM은 사용자가 입력한 Prompt를 기반으로 다음 단어를 예측하며 답변을 생성합니다.

    따라서 Prompt를 어떻게 작성하느냐에 따라 답변 품질이 크게 달라집니다.

    2. Prompt가 중요한 이유

    같은 내용을 질문하더라도 Prompt에 따라 결과가 달라집니다.

    나쁜 예

    LLM 설명해줘.

    이 Prompt는 너무 짧고 모호합니다.

    LLM 입장에서는 다음을 알 수 없습니다.

    • 누구에게 설명해야 하는지
    • 어느 정도 깊이로 설명해야 하는지
    • 어떤 형식으로 답변해야 하는지
    • 예시가 필요한지
    • 실무 관점인지 학습 관점인지

    좋은 예

    Local LLM이 무엇인지 처음 공부하는 개발자에게 설명해줘.
클라우드 기반 LLM과 비교하고, Ollama와 Open WebUI의 역할도 함께 설명해줘.
마지막에는 표로 정리해줘.

    이 Prompt는 훨씬 좋습니다.

    이유:

    • 대상이 명확함: 처음 공부하는 개발자
    • 범위가 명확함: Local LLM, 클라우드 LLM, Ollama, Open WebUI
    • 출력 형식이 명확함: 마지막에 표 정리

    3. Prompt 기본 구성 요소

    좋은 Prompt는 보통 다음 요소를 포함합니다.

    구성 요소 설명
    역할(Role) AI가 어떤 역할로 답변할지 지정
    목적(Goal) 무엇을 얻고 싶은지 명확히 지정
    배경(Context) 현재 상황이나 전제 설명
    제약조건(Constraints) 답변 길이, 난이도, 제외할 내용 등
    출력형식(Output Format) 표, 목록, 단계별 설명, 코드 등 지정

    4. 기본 Prompt 템플릿

    처음에는 아래 형식을 사용하면 좋습니다.

    너는 [역할]이다.

나는 [상황/배경]이다.

[요청사항]을 해줘.

조건:
- [조건1]
- [조건2]
- [조건3]

출력 형식:
- [원하는 형식]

    예:

    너는 AI 플랫폼 아키텍트이다.

나는 Local LLM을 처음 공부하는 개발자이다.

Ollama, Open WebUI, Local LLM 모델의 관계를 설명해줘.

조건:
- 초보자도 이해할 수 있게 설명
- 비유를 포함
- 너무 어렵게 설명하지 않기

출력 형식:
- 개념 설명
- 구성요소 표
- 전체 구조도
- 최종 정리

    http://localhost:3000

    예:

    gemma3:4b

    또는

    qwen3:8b

    그 다음 아래 Prompt를 하나씩 입력해봅니다.

    확인할 점

    7. 실습 2: 역할 부여하기

    LLM은 역할을 부여하면 답변 방향이 달라집니다.

    Prompt

    너는 AI 플랫폼 아키텍트이다.

Local LLM 기반 개발 환경을 구축할 때 필요한 구성요소를 설명해줘.

다음 항목을 포함해줘.
- Ollama
- Local LLM 모델
- Open WebUI
- Python API
- RAG 확장 가능성

출력은 아키텍처 관점으로 정리해줘.

    확인할 점

    역할을 지정하면 단순 설명보다 구조 중심 답변이 나오는지 확인합니다.

    8. 실습 3: 답변 형식 지정하기

    LLM은 출력 형식을 지정하면 더 정리된 답변을 생성합니다.

    Prompt

    Ollama와 Open WebUI의 차이를 설명해줘.

출력 형식은 반드시 아래 구조를 따라줘.

1. 한 줄 요약
2. 각각의 역할 설명
3. 차이점 비교표
4. 언제 무엇을 사용해야 하는지
5. 최종 정리

    확인할 점

    • 요청한 번호 구조를 지키는지
    • 비교표를 생성하는지
    • 결론이 명확한지

    9. 실습 4: 표로 정리하기

    Prompt

    다음 구성요소를 표로 정리해줘.

- Ollama
- Open WebUI
- Gemma
- Qwen
- Docker
- Python 가상환경

표 컬럼은 다음과 같이 만들어줘.

| 구성요소 | 역할 | 왜 필요한가 | 초보자 관점 설명 |

    확인할 점

    LLM이 표 구조를 유지하는지 확인합니다.

    10. 실습 5: 단계별 가이드 요청하기

    Prompt

    Mac에서 Ollama와 Open WebUI를 이용해 Local LLM을 구축하는 절차를 단계별로 설명해줘.

조건:
- 처음 하는 사람 기준
- 명령어 생략 금지
- 각 명령어의 의미 설명
- 실습 순서대로 작성
- 마크다운 형식으로 작성

    확인할 점

    이 Prompt는 가이드 문서 작성에 유용합니다.

    11. 실습 6: 코드 작성 요청하기

    Prompt

    Python에서 Ollama를 호출하는 가장 간단한 예제를 작성해줘.

조건:
- 모델은 gemma3:4b 사용
- 사용자 질문은 변수로 분리
- 코드에 주석 포함
- 실행 방법도 함께 설명

    예상 코드 예시

    from ollama import chat

prompt = "Local LLM이 무엇인지 설명해줘."

response = chat(
    model="gemma3:4b",
    messages=[
        {
            "role": "user",
            "content": prompt
        }
    ]
)

print(response["message"]["content"])

리뷰 기준:
- 코드 가독성

- 개선된 코드 제안

    14. 실습 9: 답변 품질 개선하기

    핵심

    좋은 답변은 좋은 Prompt에서 나옵니다.

    15. 실습 10: 답변을 재작성시키기

    LLM 답변이 너무 어렵거나 길면 다시 지시합니다.

    Prompt 예시

    방금 답변이 너무 어렵다.
초등학생도 이해할 수 있게 다시 설명해줘.

    방금 답변을 표 중심으로 다시 정리해줘.

    방금 답변을 5줄 요약으로 줄여줘.

    16. Prompt 작성 시 자주 쓰는 표현

    설명 요청

    초보자에게 설명해줘.

    비유를 들어 설명해줘.

    실무 관점에서 설명해줘.

    구조화 요청

    단계별로 설명해줘.

    표로 정리해줘.

    장점과 단점을 비교해줘.

    코드 요청

    예제 코드를 작성해줘.

    주석을 충분히 달아줘.

    실행 방법도 함께 알려줘.

    검토 요청

    문제점이 있는지 검토해줘.

    개선 방향을 제안해줘.

    실무 적용 시 주의사항을 알려줘.

    17. Prompt 품질 체크리스트

    Prompt를 작성한 뒤 아래 항목을 확인합니다.

    항목 확인 질문
    역할 AI에게 어떤 역할을 줄 것인가?
    대상 초보자, 개발자, 아키텍트 중 누구를 위한 답변인가?
    목적 설명, 비교, 코드 작성, 검토 중 무엇인가?
    범위 어디까지 설명해야 하는가?
    형식 표, 목록, 단계, 코드 중 무엇을 원하는가?
    제약 길이, 난이도, 제외 항목이 있는가?

    18. 좋은 Prompt 예시 모음

    Local LLM 학습용

    19. Local LLM Prompt 실습 시 주의사항

    Local LLM은 ChatGPT보다 다음 부분에서 부족할 수 있습니다.

    • 최신 정보 부족
    • 긴 문서 처리 능력 제한
    • 복잡한 추론 한계
    • 한국어 품질 차이
    • 모델 크기에 따른 품질 차이

    따라서 Prompt 실습 시에는 다음을 기억합니다.

    작은 모델은 짧고 명확한 지시가 좋다.
큰 모델은 복잡한 지시도 상대적으로 잘 처리한다.

    20. 모델 크기별 Prompt 전략

    모델 크기 Prompt 전략
    4B 짧고 명확하게 지시
    8B 단계별 지시 가능
    14B 실무형 설명과 코드 작성 가능
    32B 아키텍처 설계와 복잡한 비교 가능

    예를 들어 gemma3:4b에서는 한 번에 너무 많은 요구사항을 넣기보다, 단계별로 나누는 것이 좋습니다.

    21. Step1 Prompt 실습 목표

    이번 단계의 목표는 다음과 같습니다.

    • Prompt가 무엇인지 이해
    • 역할 부여 실습
    • 출력 형식 지정 실습
    • 모델별 답변 비교
    • 답변 재작성 요청
    • 코드 작성 요청
    • 문서 작성 요청
    • Local LLM의 한계 체험

    이 과정을 거치면 이후 RAG와 Agent를 구축할 때 훨씬 수월합니다.

    최종 정리

    Prompt는 LLM에게 전달하는 지시문입니다.

    좋은 Prompt는 다음을 포함합니다.

    • 역할
    • 목적
    • 배경
    • 조건
    • 출력 형식

    Local LLM을 잘 사용하려면 모델 설치만큼이나 Prompt 작성 연습이 중요합니다.
