    Local LLM 모델 크기 완전 이해 가이드 (AI 플랫폼 아키텍트 학습용)

    1. 왜 모델 크기를 이해해야 하는가?

    Local LLM을 구축할 때 가장 먼저 만나게 되는 용어가 4B, 8B, 14B, 32B, 70B 입니다.

    많은 사람들이 단순히 숫자가 클수록 좋은 모델이라고 생각하지만 실제로는 그렇지 않습니다.

    AI 플랫폼 아키텍트 관점에서는:

    • 모델 크기
    • 메모리 사용량
    • 추론 속도
    • GPU 활용
    • 운영 비용

    사이의 관계를 이해하는 것이 중요합니다.

    2. B는 무엇인가?

    B는 Billion(10억)을 의미합니다.

    예시:

    모델 파라미터 수
    4B 40억
    8B 80억
    14B 140억
    32B 320억
    70B 700억

    파라미터는 모델이 학습한 지식을 저장하는 수치입니다.

    쉽게 말하면 AI의 뇌 용량이라고 생각하면 됩니다.

    3. 파라미터(Parameter)란?

    예를 들어 사람에게

    "스프링부트가 무엇인가?"

    를 물어보면 기존 경험과 지식을 바탕으로 답변합니다.

    LLM은 학습 과정에서 수십억 개의 가중치(Weight)를 생성합니다.

    이 가중치 집합을 파라미터라고 부릅니다.

    파라미터가 많아질수록

    • 더 많은 지식 저장
    • 더 긴 문맥 이해
    • 더 복잡한 추론

    이 가능해집니다.

    4. 모델 크기별 체감 수준

    크기 체감
    1B~4B 초급 개발자
    7B~14B 경력 개발자
    20B~32B 수석 개발자
    70B 이상 전문 컨설턴트 수준

    학습용 비유이며 절대적인 기준은 아닙니다.

    5. 왜 큰 모델이 항상 좋은 것은 아닌가?

    모델이 커질수록:

    장점

    • 답변 품질 향상
    • 추론 능력 향상
    • 코드 생성 능력 향상

    단점

    • RAM 증가
    • GPU 증가
    • 응답 속도 저하
    • 다운로드 시간 증가

    6. Quantization(양자화)

    Local LLM에서 매우 중요한 개념입니다.

    원본 모델은 보통 FP16 형태입니다.

    예:

    32B 모델

    원본 크기

    약 60GB 이상

    하지만

    4bit 양자화

    적용 시

    약 18~20GB 수준까지 감소합니다.

    그래서 일반 PC에서도 실행 가능합니다.

    7. 왜 32B 모델이 32GB 메모리에서 돌아가는가?

    많은 사람들이 의아해합니다.

    원래 320억 파라미터 모델은 훨씬 큰 메모리가 필요합니다.

    하지만 Ollama는:

    • GGUF
    • Q4_K_M
    • Q5_K_M

    등의 양자화 기술을 사용합니다.

    결과:

    32B 모델도

    32GB Mac에서 실행 가능

    8. CPU와 GPU 차이

    CPU

    장점

    • 별도 장비 불필요

    단점

    • 느림

    GPU

    장점

    • 빠름

    단점

    • 고가

    9. Apple Silicon이 특별한 이유

    M2 Pro

    M3 Pro

    M4 Pro

    등은 Unified Memory 구조입니다.

    GPU와 CPU가 메모리를 공유합니다.

    따라서

    32GB Mac에서도

    14B 32B

    모델이 매우 효율적으로 동작합니다.
