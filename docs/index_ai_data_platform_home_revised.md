# AI Data Platform Project

> AI를 활용하는 개발자를 넘어, AI 플랫폼을 설계하고 구축하며 운영할 수 있는 아키텍트로 성장하기 위한 실전형 연구 프로젝트

---

## 1. 프로젝트 소개

**AI Data Platform Project**는 생성형 AI 기술을 단순히 사용하는 수준을 넘어, 기업 환경에서 실제로 활용 가능한 AI 플랫폼을 직접 설계하고 구축하기 위한 장기 연구 프로젝트입니다.

본 프로젝트는 Local LLM, RAG, AI Agent, AI Data Platform, LLM Serving, LLMOps 등 AI 서비스 구축에 필요한 핵심 기술을 단계적으로 학습하고 실습하는 것을 목표로 합니다. 이를 통해 참여자는 AI 모델의 사용법뿐만 아니라 AI 서비스가 실제 사용자에게 제공되기까지 필요한 전체 아키텍처, 데이터 흐름, 운영 체계, 보안 및 거버넌스 관점까지 함께 이해할 수 있습니다.

최근 ChatGPT를 비롯한 생성형 AI 기술은 빠르게 발전하고 있으며, 다양한 기업과 기관에서 AI 도입을 적극적으로 검토하고 있습니다. 그러나 실제 현장에서는 AI를 단순히 업무 보조 도구로 활용하는 수준에 머무르는 경우가 많고, 조직 내부 데이터와 연계하여 AI 플랫폼을 설계하고 운영할 수 있는 실무 역량은 여전히 부족한 상황입니다.

AI Data Platform Project는 이러한 현실적인 문제의식에서 출발했습니다. 우리는 AI를 사용하는 사용자에 머무르지 않고, AI 서비스를 직접 설계하고 구현하며 운영할 수 있는 구축자이자 아키텍트로 성장하는 것을 목표로 합니다.

---

## 2. 프로젝트 추진 배경

우리는 금융, 공공, 대기업 프로젝트를 수행해 온 IT 엔지니어로서 다양한 시스템 구축 경험을 보유하고 있습니다. 기존 프로젝트에서는 MSA, Cloud Native Architecture, DevOps, Middleware, API Platform, Data Integration, 데이터베이스, 보안, 운영 자동화와 같은 기술을 활용해 왔습니다.

그러나 AI 기술은 기존 엔터프라이즈 시스템 구축 경험만으로는 충분히 대응하기 어려운 새로운 영역입니다. AI 서비스는 모델, 데이터, 검색, 추론, API, 사용자 인터페이스, 보안, 운영 모니터링이 유기적으로 결합되어야 하며, 특히 기업 내부 데이터를 안전하게 활용하기 위한 별도의 아키텍처 설계가 필요합니다.

현재 시장에서는 다음과 같은 요구가 빠르게 증가하고 있습니다.

- 기업 내부 데이터 기반 AI 서비스 구축
- 사내 AI 비서 및 업무 지원 Agent 구축
- RAG 기반 문서 검색 및 지식 검색 서비스 구축
- On-Premise 기반 Local LLM 운영 환경 구축
- AI Agent 기반 업무 자동화 체계 구축
- AI Data Platform 및 데이터 파이프라인 구축
- LLM Serving 및 LLMOps 운영 체계 수립
- AI Governance, 보안, 권한 관리, 감사 체계 수립

대부분의 AI 학습은 이론 학습이나 외부 서비스 사용법 중심으로 진행되는 경우가 많습니다. 하지만 실제 기업 프로젝트에서는 AI 모델 자체보다도 데이터를 어떻게 준비하고, 어떤 구조로 검색하며, 어떤 방식으로 API와 연계하고, 어떻게 운영·관리할 것인지가 더욱 중요합니다.

본 프로젝트는 이러한 한계를 극복하기 위해 시작되었습니다. 우리는 단순한 기술 학습을 넘어 실제 구축 가능한 수준의 아키텍처, 실습 코드, 운영 문서, 팀 공유 자산을 함께 만들어 나가고자 합니다.

---

## 3. 프로젝트 비전

본 프로젝트의 비전은 다음과 같습니다.

> **AI를 이해하고, AI를 구축하며, AI를 운영할 수 있는 엔지니어로 성장한다.**

우리는 특정 AI 모델이나 도구의 사용법을 익히는 것에 그치지 않습니다. AI 서비스가 실제 업무 환경에서 안정적으로 동작하기 위해 필요한 기술 요소를 전체 관점에서 이해하고, 이를 직접 구현할 수 있는 역량을 확보하는 것을 목표로 합니다.

이를 위해 다음 영역을 단계적으로 학습하고 실습합니다.

- Local LLM 구축 및 운영
- Prompt Engineering 및 Open WebUI 활용
- RAG 구조 설계 및 Vector Database 활용
- AI Agent 및 Tool Calling 구조 설계
- 데이터 수집, 적재, 처리, 분석을 위한 AI Data Platform 설계
- LLM Serving 및 API 기반 서비스 제공 구조 설계
- LLMOps, 모니터링, 보안, 거버넌스 체계 학습

## 4. 연구 그룹 소개

본 프로젝트는 **알앤비소프트 소속 엔지니어들이 자발적으로 구성한 AI 연구 그룹**에서 추진하고 있습니다.

연구 그룹은 공식 조직 체계와 별개로 운영되는 자율 연구 조직이며, 각 참여자가 보유한 시스템 구축 경험과 기술 역량을 바탕으로 AI 플랫폼 기술을 함께 학습하고 검증하는 것을 목적으로 합니다.

우리의 활동 방향은 다음과 같습니다.

- 실습 중심의 기술 학습을 수행합니다.
- 학습 내용을 문서화하여 조직 내 지식 자산으로 축적합니다.
- 실제 프로젝트에 적용 가능한 아키텍처와 구현 방식을 검토합니다.
- 팀원이 동일한 환경에서 실습할 수 있도록 코드와 가이드를 함께 관리합니다.
- 장기적으로 AI 기반 서비스와 플랫폼을 직접 제안하고 구축할 수 있는 역량을 확보합니다.

본 연구 그룹은 AI를 일시적인 기술 트렌드로 바라보지 않습니다. 앞으로의 IT 프로젝트에서 AI가 핵심 기술 요소로 자리 잡을 것으로 판단하고, 이에 대응하기 위한 실무형 역량을 선제적으로 확보하고자 합니다.

## 5. 전체 로드맵

AI Data Platform Project는 단계별 학습과 실습을 통해 점진적으로 확장됩니다.

### STEP 1. Local LLM 구축

Local LLM 환경을 구축하고, 로컬 환경에서 AI 모델을 실행하는 기본 구조를 이해합니다.

주요 학습 항목은 다음과 같습니다.

- Ollama 설치 및 모델 실행
- Open WebUI 구축 및 활용
- Prompt Engineering 기본 실습
- 로컬 LLM API 호출 구조 이해
- Docker 기반 실행 환경 이해

### STEP 2. RAG 구축

내부 문서와 데이터를 기반으로 검색 증강 생성 구조를 구축합니다.

주요 학습 항목은 다음과 같습니다.

- 문서 수집 및 전처리
- Chunking 전략 수립
- Embedding 모델 활용
- Vector Database 구축
- Retrieval 구조 설계
- RAG 기반 질의응답 흐름 구현

### STEP 3. AI Agent 구축

LLM이 외부 도구와 시스템을 활용하여 작업을 수행할 수 있는 Agent 구조를 학습합니다.

주요 학습 항목은 다음과 같습니다.

- Tool Calling 구조 이해
- Workflow 기반 Agent 설계
- Memory 구조 활용
- Multi-Agent 구조 검토
- 업무 자동화 시나리오 설계

### STEP 4. AI Data Platform 구축

AI 서비스에 필요한 데이터를 수집, 저장, 처리, 분석하기 위한 데이터 플랫폼 구조를 학습합니다.

주요 학습 항목은 다음과 같습니다.

- Data Lake 구조 이해
- Data Warehouse 구조 이해
- Lakehouse 아키텍처 검토
- CDC 기반 데이터 동기화
- Streaming 데이터 처리
- 메타데이터 및 데이터 거버넌스

### STEP 5. LLM Serving Platform 구축

AI 모델을 서비스 형태로 제공하기 위한 Serving 및 운영 구조를 학습합니다.

주요 학습 항목은 다음과 같습니다.

- vLLM 기반 Serving 구조
- GPU 기반 추론 환경 이해
- API Gateway 연계 구조
- 인증 및 권한 관리
- LLMOps 운영 체계
- 모니터링 및 성능 관리

## 7. 향후 연구 과제

AI Data Platform Project는 기본 AI 플랫폼 기술 학습을 넘어, 향후 금융 및 투자 데이터 영역으로 확장하는 것을 목표로 합니다.

특히 AI 기술을 실제 증권 및 금융 서비스에 접목하여 다음과 같은 서비스를 연구할 계획입니다.

- AI 투자 비서
- 종목 분석 Agent
- 뉴스 분석 Agent
- 공시 분석 Agent
- 포트폴리오 분석 Agent
- Multi-Agent 기반 Trading Platform

이러한 서비스는 초기 단계에서 바로 자동매매를 목표로 하지 않습니다. 우선 LLM과 RAG를 활용하여 공시, 뉴스, 리포트, 재무 데이터 등 다양한 정보를 분석하고, 관심종목 후보를 생성하는 의사결정 지원 서비스부터 단계적으로 접근합니다.

---

## 8. 금융 데이터 기반 AI 서비스 확장 로드맵

### STEP 1. RAG 기반 공시 이벤트 종목 스크리너

공시 데이터를 수집하고, 주요 이벤트를 분석하여 관심종목 후보를 도출하는 구조를 연구합니다.

주요 연구 항목은 다음과 같습니다.

- OpenDART 등 공시 데이터 수집
- 공급계약, 신규시설투자, 자사주 취득, 실적 개선, 배당 확대 등 긍정 이벤트 식별
- 횡령·배임, 감사의견 비적정, 관리종목 지정, 대규모 전환사채, 반복적인 유상증자 등 부정 이벤트 식별
- 공시 원문 Vector DB 저장 및 RAG 검색 구조 구성
- LLM 기반 공시 이벤트 요약, 근거 추출, 리스크 정리
- 관심종목 후보, 제외종목 후보, 추가 확인 필요 종목 분류

### STEP 2. 뉴스 및 리포트 기반 종목 분석 Agent

종목별 뉴스, 산업 뉴스, 증권사 리포트, 실적 발표 자료를 수집하고 분석하는 Agent를 구축합니다.

주요 연구 항목은 다음과 같습니다.

- 종목별 뉴스 및 산업 뉴스 수집
- 증권사 리포트 및 실적 발표 자료 분석
- 정책 수혜, 테마 관련성, 사업 연관성 검토
- 단순 키워드 언급이 아닌 실제 매출, 수주, 투자계획과의 연결성 확인
- 종목별 핵심 이슈 및 리스크 요약

### STEP 3. 정량 데이터 기반 종목 스코어링

공시와 뉴스에서 추출한 정성 데이터와 가격, 거래량, 재무지표 등 정량 데이터를 결합하여 종목별 점수를 산출합니다.

주요 연구 항목은 다음과 같습니다.

- 가격, 거래량, 거래대금 데이터 수집
- 재무지표 및 수급 데이터 수집
- 모멘텀, 가치, 퀄리티, 수급, 변동성 팩터 계산
- RAG 분석 점수와 정량 팩터 점수 결합
- 종목별 종합 점수 산출 및 우선순위 선정

### STEP 4. AI 기반 종목 후보 추출 서비스

RAG 분석 결과와 정량 점수를 기반으로 매일 또는 매주 관심종목 후보를 추출하는 서비스를 구성합니다.

주요 연구 항목은 다음과 같습니다.

- 관심종목 후보 자동 생성
- 종목별 선정 사유 제공
- 핵심 공시, 긍정 요인, 부정 요인, 리스크 제공
- 리서치 리포트 형태의 결과 화면 구성
- 매수, 관망, 제외, 추가 확인 필요 상태 관리

### STEP 5. 백테스트 및 전략 검증

과거 데이터를 기준으로 종목 추출 전략의 유효성을 검증합니다.

주요 연구 항목은 다음과 같습니다.

- 과거 공시, 뉴스, 가격 데이터 기반 전략 검증
- 리밸런싱 주기, 보유 기간, 종목 수, 비중 설정
- 거래비용 및 슬리피지 반영
- 누적수익률, 연평균수익률, MDD, 승률, 손익비 측정
- 시장 대비 초과수익률 분석
- RAG 이벤트 기준 및 정량 팩터 비중 개선

### STEP 6. 모의투자 및 성과 모니터링

실거래 전 모의투자를 통해 실제 시장 환경에서 전략의 성과를 검증합니다.

주요 연구 항목은 다음과 같습니다.

- 후보 종목의 1주, 2주, 4주 후 성과 추적
- 백테스트 결과와 모의투자 결과 비교
- 신호 지연, 과열 종목 추격, 유동성 부족 여부 점검
- 리스크 이벤트 누락 여부 점검
- 전략 개선 및 운영 기준 보완

### STEP 7. 트레이딩 플랫폼 확장

모의투자 성과가 안정적으로 검증된 이후, 단계적으로 트레이딩 플랫폼으로 확장합니다.

주요 연구 항목은 다음과 같습니다.

- 초기 단계의 반자동 투자 지원 구조 적용
- AI가 후보를 제시하고 사람이 최종 판단하는 구조 설계
- 포트폴리오 비중 산정
- 리밸런싱 기준 수립
- 손절 및 익절 기준 검토
- 주문 연동 기능 단계적 추가
- Multi-Agent 기반 Trading Platform 확장

## 9. 최종 목표

AI Data Platform Project의 최종 목표는 AI를 단순히 사용하는 개발자가 아니라, AI 플랫폼을 설계하고 구축할 수 있는 아키텍트로 성장하는 것입니다.

우리는 본 프로젝트를 통해 AI 서비스 구축에 필요한 기술 요소를 단계적으로 검증하고, 실습 결과를 문서와 코드로 축적하며, 실제 비즈니스에 적용 가능한 AI 플랫폼 구축 역량을 확보하고자 합니다.

궁극적으로는 기업 내부 데이터와 AI 기술을 결합하여 업무 생산성을 높이고, 금융 데이터 기반 AI 서비스와 AI 투자 지원 플랫폼으로 확장함으로써 실질적인 비즈니스 가치를 창출하는 것을 목표로 합니다.
