# Step1-5 Prompt 실습 가이드

## 문서 목적

본 문서는 Local LLM 구축 Step1 과정 중 **Prompt 실습**을 위한 가이드입니다.

이전 단계에서 우리는 다음 작업을 완료했습니다.

```text
Step1-1 Ollama 설치
Step1-2 Local LLM 모델 실행
Step1-3 Python 가상환경과 Ollama 연동
Step1-4 Open WebUI 구축
```

이제부터는 Open WebUI 또는 Ollama API를 통해 Local LLM에게 질문을 잘 전달하는 방법을 연습합니다.

Prompt 실습의 목적은 단순히 질문을 던지는 것이 아니라, **LLM이 원하는 방식으로 답변하도록 지시하는 방법**을 익히는 것입니다.

---

## 1. Prompt란 무엇인가?

Prompt는 LLM에게 전달하는 입력 문장입니다.

쉽게 말하면 다음과 같습니다.

```text
Prompt = AI에게 주는 지시문
```

예:

```text
Local LLM이 무엇인지 초보자에게 설명해줘.
```

이 문장이 바로 Prompt입니다.

LLM은 사용자가 입력한 Prompt를 기반으로 다음 단어를 예측하며 답변을 생성합니다.

따라서 Prompt를 어떻게 작성하느냐에 따라 답변 품질이 크게 달라집니다.

---

## 2. Prompt가 중요한 이유

같은 내용을 질문하더라도 Prompt에 따라 결과가 달라집니다.

### 나쁜 예

```text
LLM 설명해줘.
```

이 Prompt는 너무 짧고 모호합니다.

LLM 입장에서는 다음을 알 수 없습니다.

- 누구에게 설명해야 하는지
- 어느 정도 깊이로 설명해야 하는지
- 어떤 형식으로 답변해야 하는지
- 예시가 필요한지
- 실무 관점인지 학습 관점인지

---

### 좋은 예

```text
Local LLM이 무엇인지 처음 공부하는 개발자에게 설명해줘.
클라우드 기반 LLM과 비교하고, Ollama와 Open WebUI의 역할도 함께 설명해줘.
마지막에는 표로 정리해줘.
```

이 Prompt는 훨씬 좋습니다.

이유:

- 대상이 명확함: 처음 공부하는 개발자
- 범위가 명확함: Local LLM, 클라우드 LLM, Ollama, Open WebUI
- 출력 형식이 명확함: 마지막에 표 정리

---

## 3. Prompt 기본 구성 요소

좋은 Prompt는 보통 다음 요소를 포함합니다.

| 구성 요소 | 설명 |
|---|---|
| 역할(Role) | AI가 어떤 역할로 답변할지 지정 |
| 목적(Goal) | 무엇을 얻고 싶은지 명확히 지정 |
| 배경(Context) | 현재 상황이나 전제 설명 |
| 제약조건(Constraints) | 답변 길이, 난이도, 제외할 내용 등 |
| 출력형식(Output Format) | 표, 목록, 단계별 설명, 코드 등 지정 |

---

## 4. 기본 Prompt 템플릿

처음에는 아래 형식을 사용하면 좋습니다.

```text
너는 [역할]이다.

나는 [상황/배경]이다.

[요청사항]을 해줘.

조건:
- [조건1]
- [조건2]
- [조건3]

출력 형식:
- [원하는 형식]
```

예:

```text
너는 AI 플랫폼 아키텍트이다.

나는 Local LLM을 처음 공부하는 개발자이다.

Ollama, Open WebUI, Local LLM 모델의 관계를 설명해줘.

조건:
- 초보자도 이해할 수 있게 설명
- 비유를 포함
- 너무 어렵게 설명하지 않기

출력 형식:
- 개념 설명
- 구성요소 표
- 전체 구조도
- 최종 정리
```

## 11. 실습 6: 코드 작성 요청하기

### Prompt

```text
Python에서 Ollama를 호출하는 가장 간단한 예제를 작성해줘.

조건:
- 모델은 gemma3:4b 사용
- 사용자 질문은 변수로 분리
- 코드에 주석 포함
- 실행 방법도 함께 설명
```

### 예상 코드 예시

```python
from ollama import chat

prompt = "Local LLM이 무엇인지 설명해줘."

response = chat(
    model="gemma3:4b",
    messages=[
        {
            "role": "user",
            "content": prompt
        }
    ]
)

print(response["message"]["content"])
```

## 19. Local LLM Prompt 실습 시 주의사항

Local LLM은 ChatGPT보다 다음 부분에서 부족할 수 있습니다.

- 최신 정보 부족
- 긴 문서 처리 능력 제한
- 복잡한 추론 한계
- 한국어 품질 차이
- 모델 크기에 따른 품질 차이

따라서 Prompt 실습 시에는 다음을 기억합니다.

```text
작은 모델은 짧고 명확한 지시가 좋다.
큰 모델은 복잡한 지시도 상대적으로 잘 처리한다.
```

## 20. 모델 크기별 Prompt 전략

| 모델 크기 | Prompt 전략 |
|---|---|
| 4B | 짧고 명확하게 지시 |
| 8B | 단계별 지시 가능 |
| 14B | 실무형 설명과 코드 작성 가능 |
| 32B | 아키텍처 설계와 복잡한 비교 가능 |

예를 들어 `gemma3:4b`에서는 한 번에 너무 많은 요구사항을 넣기보다, 단계별로 나누는 것이 좋습니다.
